## 目錄結構

```
.
├── raw/          # 原始資料（唯讀，LLM 只讀不改）
│   └── assets/   # 圖片附件
├── wiki/         # LLM 生成與維護的知識頁面
│   ├── index.md  # 頁面總目錄（每次 ingest 後更新）
│   ├── log.md    # 操作紀錄（append-only）
│   ├── overview.md       # 課程整體概覽與核心論點
│   ├── concepts/         # 概念頁面（如「情緒調節」、「共情」）
│   ├── sessions/         # 各堂課摘要頁面
│   └── syntheses/        # 跨堂課的分析、比較、綜合
└── CLAUDE.md     # 本 schema 文件
```

**規則：**
- `raw/` 內容永遠不修改。
- `wiki/` 由 LLM 全權負責建立與更新。
- 每個 wiki 頁面都使用 YAML frontmatter（見下方格式）。

## Wiki 頁面格式

```markdown
---
title: 頁面標題
type: concept | session | synthesis | overview
tags: []
sources: []       # 引用的 raw/ 檔案名稱
updated: YYYY-MM-DD
---

# 頁面標題

內容...
```

**連結慣例：** 使用 Obsidian wiki link 格式 `[[頁面名稱]]` 進行頁面間的交叉引用。

## 操作流程

### Ingest（消化新來源）

當我說「消化 [檔案名]」或「ingest [檔案名]」時：

1. 讀取 `raw/[檔案名]`
2. 與我討論重點摘要（可選）
3. 在 `wiki/sessions/` 建立該堂課的摘要頁面
4. 更新 `wiki/concepts/` 中相關概念頁面（新增或補充）
5. 更新 `wiki/overview.md` 的整體論點（如有新洞見）
6. 更新 `wiki/index.md`
7. 在 `wiki/log.md` 追加一筆記錄：`## [YYYY-MM-DD] ingest | 標題`

一次 ingest 通常會觸及 5–15 個頁面，這是正常的。

### Query（查詢）

當我提問時：

1. 先讀 `wiki/index.md` 找相關頁面
2. 讀取相關頁面，綜合回答
3. 引用具體頁面作為來源
4. 若答案有新洞見值得留存，詢問我是否要存成 `wiki/syntheses/` 頁面

### Lint（健檢）

當我說「lint wiki」時，檢查：

- 矛盾：不同頁面間有衝突的說法
- 孤立頁面：沒有任何 inbound 連結的頁面
- 缺口：重要概念被提及但沒有自己的頁面
- 過時：早期摘要被新資料推翻但未更新
- 建議：可以去尋找的新資料來源

## 情緒教養課 特定慣例

### 概念頁面命名
使用繁體中文，例如：`wiki/concepts/情緒調節.md`、`wiki/concepts/共情.md`

### 課堂摘要頁面命名
格式：`wiki/sessions/[單元編號]-[主題].md`，例如：`wiki/sessions/01-情緒的本質.md`

### 核心追蹤維度
每堂課摘要頁面應包含：
- **本堂核心主張**：這堂課最重要的一個觀點
- **關鍵概念**：本堂出現的重要術語（連結到 concepts/ 頁面）
- **實用技巧**：可直接應用的方法或練習
- **延伸連結**：與其他堂課或概念的關聯

## index.md 格式

```markdown
# Wiki 索引

## 課堂摘要 (sessions/)
- [[session-page]] — 一行摘要

## 概念 (concepts/)
- [[concept-page]] — 一行定義

## 綜合分析 (syntheses/)
- [[synthesis-page]] — 一行說明

## 課程概覽
- [[overview]] — 整體架構與核心論點
```

## log.md 格式

每筆記錄使用固定前綴，方便 grep：

```
## [YYYY-MM-DD] ingest | 來源標題
## [YYYY-MM-DD] query | 問題摘要
## [YYYY-MM-DD] lint | 健檢摘要
```
